### More Effective C# 비동기 원칙

```
┌─────────────────────────────────────────────────────────────────┐
│                    비동기 프로그래밍 원칙                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 동기 API와 비동기 API를 함께 제공하지 마라                   │
│  2. async void를 피하라 (이벤트 핸들러 제외)                    │

│  5. ValueTask를 적절히 활용하라                                 │
│  6. 병렬 처리가 필요하면 Task.WhenAll을 사용하라                │
│  7. 비동기 코드에서 lock 대신 SemaphoreSlim을 사용하라          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### ASP.NET Core 가이드라인

```
┌─────────────────────────────────────────────────────────────────┐
│                    ASP.NET Core 원칙                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  DI (의존성 주입):                                              │
│  • Singleton에서 Scoped 서비스를 직접 주입하지 마라              │
│  • 서비스 로케이터 패턴을 피하라                                 │
│  • 생성자 주입을 선호하라                                       │
│                                                                 │
│  성능:                                                          │
│  • HttpClient를 직접 생성하지 말고 IHttpClientFactory 사용       │

│  • sync-over-async를 피하라 (.Result, .Wait())                  │
│                                                                 │
│  보안:                                                          │
│  • 민감 정보를 로그에 남기지 마라                               │
│  • 사용자 입력을 항상 검증하라                                  │
│  • HTTPS를 항상 사용하라                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
